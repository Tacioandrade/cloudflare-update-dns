# 変更履歴

GitHub のプロジェクト: [https://github.com/Tacioandrade/cloudflare-update-dns/](https://github.com/Tacioandrade/cloudflare-update-dns/)

このプロジェクトの主な変更はすべてこのファイルに記録されます。

## [2.0.0] - 2026-07-14
### 追加 / 変更
- ドメイン一覧は、Cloudflare API から各ドメインを受信すると、すべてのページの完了を待たずに順次表示されるようになりました。
- ポルトガル語、英語、フランス語、スペイン語、簡体字中国語、日本語に対応し、選択した言語の保存とシステム言語の検出を追加しました。
- バージョン履歴が、アプリケーションで選択した言語で表示されるようになりました。

## [1.1.4] - 2026-06-27
### 追加 / 変更
- プロジェクト名を Cloudflare Update DNS から Cloudflare DNS Manager に変更しました。
- 新しい DNS レコードの作成画面を開くキーボードショートカット `Ctrl + N` を追加しました。

## [1.1.3] - 2026-06-26
### 修正
- Microsoft Visual C++ Runtime がないことに関連する Windows 版の実行時エラーを修正しました。
- 現在のバージョンの変更履歴のみを公開し、Windows と Linux のバンドルを `cloudflare-update-dns` フォルダー内にパッケージ化するよう、プロジェクトの CI/CD 設定を修正しました。

## [1.1.2] - 2026-06-26
### 追加 / 修正
- DNS レコードの作成および編集時に入力内容を検証する機能を追加しました。
- `A` レコードでは IPv4、`AAAA` レコードでは IPv6 を検証するようにしました。
- 固定された TLD リストに依存せず、`CNAME`、`MX`、`NS` レコードのドメイン形式を検証するようにしました。
- `SRV` レコードの基本形式と、`TXT` レコードの内容が空でないことを検証するようにしました。
- `CNAME`、`MX`、`NS` レコードおよび `SRV` のターゲットに入力された IP アドレスを拒否するよう、ドメイン検証を修正しました。
- DMARC や DKIM などの `TXT` レコード作成時に、プロキシ非対応タイプへ `proxied` フィールドを誤って送信しないよう修正しました。
- `Proxied` コントロールは、`A`、`AAAA`、`CNAME` レコードでのみ表示されるようになりました。
- Cloudflare API がレコードの作成または更新を拒否した場合に表示されるエラーメッセージを調整しました。
- Windows と Linux のアプリケーションを自動生成するプロジェクトの CI/CD を有効にしました。

## [1.1.1] - 2026-06-25
### 追加 / 修正
- アプリケーションを開いた際、設定された認証方式に応じてパスワードまたは生体認証フィールドにキーボードフォーカスが置かれるようになりました。
- ドメイン一覧と DNS レコード一覧の検索をすばやく開くキーボードショートカット `Ctrl + F` を追加しました。
- 検索機能をすばやく閉じるキーボードショートカット `ESC` を追加しました。
- DNS レコードの作成/編集画面をすばやく閉じるキーボードショートカット `ESC` を追加しました。
- DNS レコードの作成/編集画面または設定画面からドメイン一覧に戻るキーボードショートカット `ESC` を追加しました。
- ログイン画面のパスワードフィールドで `Enter` キーを押すと自動的にログインする機能を実装しました。
- 作成/編集中にサブドメインと内容のフィールドで `Enter` キーを押すと DNS レコードを自動保存する機能を実装しました。

## [1.1.0] - 2026-06-25
### 追加
- Windows 版のドキュメントとビルド/テスト手順を追加しました。
- Linux 版のドキュメントとビルド/テスト手順を追加しました。
### セキュリティ
- Cloudflare API Token の保存先を `flutter_secure_storage` に変更し、各プラットフォームのネイティブな安全なストレージを使用するようにしました。
- アプリのパスワードを、安全なストレージ内でランダムなソルトを用いた PBKDF2-HMAC-SHA256 ハッシュとして保存するよう変更し、平文パスワードの永続化を廃止しました。
- 認証済みセッションはメモリ内でのみ維持し、アプリケーションを再度開いた際に新たなログインを要求するようにしました。
- テーマや DNS レコードタイプなど、機密でない設定のみを `shared_preferences` に保持するようにしました。

## [1.0.8] - 2026-06-24
### 修正
- アプリケーションを閉じて再度開いた際に強制的にログアウトし、すべてのオペレーティングシステムで再ログインを要求するようにしました。

## [1.0.7] - 2026-06-24
### 追加 / 修正
- オペレーティングシステムに設定されたテーマを自動検出し、ライトテーマとダークテーマを使用できるようにしました。
- 設定でシステム、ライト、ダークの各テーマを選択できるオプションを追加しました。
- コピー、貼り付け、すべて選択などの内部テキストをデバイスの言語で表示する Flutter ネイティブのローカライズを追加しました。

## [1.0.6] - 2026-06-24

- カスタム Cloudflare Token で CDN キャッシュ削除を使用するために必要な権限を文書化しました。

## [1.0.5] - 2026-06-24

- 初回アクセス時に、既定のパスワードや生体認証を要求せず、パスワード作成画面を直接開くように変更しました。
- ログイン画面からユーザーフィールドを削除し、登録済みパスワードまたは生体認証のみでアクセスするようにしました。

## [1.0.4] - 2026-06-19

- アプリケーション内で変更履歴を直接表示する画面を追加しました。

## [1.0.3] - 2026-06-19
### 追加 / 修正
- 透過的かつ自動的なフォールバックを備えたネイティブ生体認証ログイン（指紋 / 顔認識）を実装しました。

## [1.0.2] - 2026-06-18
### 追加 / 修正
- API 連携ロジックを修正し、自動ページネーションを実行するようにしました。アプリケーションは、Cloudflare の既定値である1ページあたり20レコードの制限を回避し、トークンがアクセスできるすべてのドメインと DNS レコードを取得するようになりました。
- インターフェースとビジュアルアイデンティティを細かく調整しました。
- 新しいロゴを実装し、標準化されたアイコン（Android、iOS、Web）を生成しました。

## [1.0.1] - 2026-06-18

- 安全にコンパイルできるよう、Kotlin のバージョンと NDK 設定を調整しました。
- パッケージ識別子（Application ID）をサンプルの既定値から正式な企業署名へ変更しました。
